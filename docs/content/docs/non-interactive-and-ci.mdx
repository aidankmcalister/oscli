This page covers the runtime behavior that matters in CI, scripts, and piped
output. These rules are automatic. You do not need a separate mode switch.

## TTY detection

`oscli` checks stdout at runtime and switches behavior when output is not a TTY.
That covers CI, pipes, and test harnesses.

<Callout type="warn">
  In non-TTY mode, prompts use matching flags, defaults, or optional values.
  If a required prompt has no value, `oscli` exits with usage code `2`. Spinner
  animation stops, progress prints sequential lines, and ANSI styling is
  stripped automatically.
</Callout>

## Semantic exit codes

Use named exit codes when other tooling needs to branch on failure type.

```ts title="src/cli.ts"
cli.exit("Not authenticated.", { code: "auth" });
cli.exit("File not found.", { code: "not_found" });
cli.exit("Network request failed.", { code: "network" });
```

## Autocomplete hint

If you expose a completion command elsewhere, you can guide users when they hit
unknown commands or flags.

```ts title="src/cli.ts"
export const cli = createCLI(() => ({
  description: "workspace tool",
  autocompleteHint: "Run `oscli completion` to enable tab completion",
  prompts: {},
}));
```

That hint prints as a secondary line under the main error.
