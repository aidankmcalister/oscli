`oscli` is a TypeScript-first CLI framework for Bun. You define prompts and
flags once with a builder API, run your flow, then read typed values from
`cli.storage` and `cli.flags`.

## Install

Use the package manager you already use.

```package-install title="Install oscli"
@oscli-dev/oscli
```

## Your first CLI

Start with the smallest possible CLI. This example stays under 10 lines and is
complete.

```ts title="src/cli.ts"
import { createCLI } from "@oscli-dev/oscli";

const cli = createCLI(() => ({ description: "my tool", prompts: {} }));

await cli.run(async () => {
  cli.success("Done.");
});
```

Use this shape when you want a single-command CLI with no prompts yet.

## Build a prompt-driven CLI

Add prompts, run them once, and read the resolved values from `cli.storage`.

```ts title="src/cli.ts"
import { createCLI } from "@oscli-dev/oscli";

const cli = createCLI((b) => ({
  description: "project setup",
  prompts: {
    project: b.text().label("Project").default("my-app"),
    framework: b
      .select({ choices: ["react", "vue", "svelte"] as const })
      .label("Framework"),
    approved: b.confirm().label("Continue?").default(true),
  },
}));

await cli.run(async () => {
  cli.intro("project setup");
  await cli.prompt.project();
  await cli.prompt.framework();
  await cli.prompt.approved();

  cli.success(`Created ${cli.storage.project}`);
  cli.outro("Done.");
});
```

## Grow the same CLI

You can add JSON mode and return a machine-readable result without changing the
rest of the flow.

```ts title="src/cli.ts"
const cli = createCLI((b) => ({
  description: "project setup",
  prompts: {
    project: b.text().label("Project"),
  },
  json: true, // [!code ++]
}));

await cli.run(async () => {
  await cli.prompt.project();
  cli.setResult({ project: cli.storage.project }); // [!code ++]
});
```

Run that command with `--json` to suppress decorative output and print only the
JSON result.

## Setup flow

Use this order when you are starting a new CLI.

<Steps>
  <Step>
    <h3>Install the package</h3>
    <p>
      Add <code>@oscli-dev/oscli</code> to your project.
    </p>
  </Step>
  <Step>
    <h3>Define prompts and flags</h3>
    <p>
      Put your builder definitions inside <code>createCLI()</code> so types are
      inferred once and reused everywhere.
    </p>
  </Step>
  <Step>
    <h3>Run the flow</h3>
    <p>
      Call <code>await cli.run(...)</code>, resolve prompts, and read values
      from <code>cli.storage</code> and <code>cli.flags</code>.
    </p>
  </Step>
</Steps>

<Callout type="info">
  <code>oscli</code> is Bun-native, but the published package works in Node 18+
  at runtime because the output bundle targets Node.
</Callout>
