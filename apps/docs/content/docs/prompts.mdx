`oscli` prompt builders are the center of the framework. You define the input
shape once, resolve prompts with `cli.prompt.<name>()`, then read typed values
from `cli.storage`.

## Canonical chain order

Use the same chain order for every prompt builder so definitions stay easy to
scan.

```ts title="Prompt chain order"
b.<type>()
  .label()
  .describe()
  .placeholder()
  .default()
  .optional()
  .validate()
  .transform()
  .color()
```

## Text

Use `b.text()` for plain string input.

```ts title="src/cli.ts"
export const cli = createCLI((b) => ({
  title: "text prompt",
  prompts: {
    project: b
      .text()
      .label("Project")
      .placeholder("my-app")
      .default("my-app")
      .validate((value) =>
        value.length >= 2 ? true : "Use at least 2 characters.",
      )
      .transform((value) => value.trim()),
  },
}));
```

```oscli title="Prompting"
Project
› my-app_
```

```oscli title="Submitted"
✓ Project: my-app
```

## Password

Use `b.password()` when the user types sensitive text.

```ts title="src/cli.ts"
export const cli = createCLI((b) => ({
  title: "password prompt",
  prompts: {
    token: b.password().label("API token"),
  },
}));
```

```oscli title="Prompting"
API token
› ********_
```

```oscli title="Submitted"
✓ API token: ********
```

## Number

Use `b.number()` when the stored value must be a number.

```ts title="src/cli.ts"
export const cli = createCLI((b) => ({
  title: "number prompt",
  prompts: {
    retries: b.number().label("Retries").min(0).max(5),
    budget: b.number().label("Budget").prefix("$").min(0),
  },
}));
```

```oscli title="Prompting"
Budget
$ › 500_
```

```oscli title="Submitted"
✓ Budget: 500
```

## Select

Use `b.select()` for one choice from a fixed list.

```ts title="src/cli.ts"
export const cli = createCLI((b) => ({
  title: "select prompt",
  prompts: {
    mode: b
      .select({ choices: ["personal", "team"] as const })
      .label("Mode")
      .rule("personal", "single-user setup")
      .rule("team", "shared project"),
  },
}));
```

```oscli title="Prompting"
Mode
› ● personal   single-user setup
  ○ team       shared project
↑↓ navigate   enter select
```

```oscli title="Submitted"
✓ Mode: personal
```

## Multiselect

Use `b.multiselect()` when the user can choose multiple values.

```ts title="src/cli.ts"
export const cli = createCLI((b) => ({
  title: "multiselect prompt",
  prompts: {
    features: b
      .multiselect({ choices: ["api", "ui", "docs", "tests"] as const })
      .label("Features")
      .min(1)
      .max(3),
  },
}));
```

```oscli title="Prompting"
Features  (1–3)
› ◉ api
  ○ ui
  ◉ docs
  ○ tests
↑↓ navigate   space toggle   enter confirm
```

```oscli title="Submitted"
✓ Features: api, docs
```

## Confirm

Use `b.confirm()` for boolean input. Toggle is the default. Simple keeps the
older typed `y/n` flow.

<Tabs items={["Toggle", "Simple"]}>
  <Tab value="Toggle">

```ts title="src/cli.ts"
approved: b.confirm().label("Continue?").default(true)
```
  </Tab>
  <Tab value="Simple">

```ts title="src/cli.ts"
approved: b.confirm("simple").label("Continue?").default(false)
```
  </Tab>
</Tabs>

- `Toggle`: inline `Yes / No` selector, arrow keys or `y`/`n`, then Enter
- `Simple`: typed `y/n` input

```oscli title="Toggle prompting"
Continue?
● Yes  /  ○ No
```

```oscli title="Toggle submitted"
✓ Continue?: yes
```

```oscli title="Simple prompting"
Continue?
› (y/N) _
```

```oscli title="Simple submitted"
✓ Continue?: no
```

## Search

Use `b.search()` when the choice list is fixed, but the user needs to filter it
while typing.

```ts title="src/cli.ts"
export const cli = createCLI((b) => ({
  title: "search prompt",
  prompts: {
    framework: b
      .search({ choices: ["react", "vue", "svelte", "solid"] as const })
      .label("Framework")
      .rule("react", "large ecosystem")
      .rule("svelte", "minimal runtime"),
  },
}));
```

```oscli title="Prompting"
Framework
› sv_
› ● svelte   minimal runtime
  ○ solid
↑↓ navigate   enter select
```

```oscli title="Submitted"
✓ Framework: svelte
```

## List

Use `b.list()` for repeated free-form input. The prompt ends on an empty Enter.

export const cli = createCLI((b) => ({
  title: "list prompt",

    tags: b.list().label("Tags").min(1).max(5),

## Date

Use `b.date()` for validated date input stored as `Date`.

```ts title="src/cli.ts"
export const cli = createCLI((b) => ({
  title: "date prompt",
  prompts: {
    deadline: b.date().label("Deadline").format("YYYY-MM-DD"),
  },
}));
```

```oscli title="Prompting"
Deadline
› 2026-03-31_
```

```oscli title="Submitted"
✓ Deadline: 2026-03-31
```

```oscli title="Validation error"
Deadline
› tomorrow_
✗ Use a valid date in YYYY-MM-DD format.
```

<Callout type="info">
  If you do not set a placeholder for `b.date()`, `oscli` uses the configured
  format as the placeholder.
</Callout>

## Resolution behavior

Prompt builders share the same runtime rules once a value is submitted.

<Accordions type="multiple">
  <Accordion title="Validation and transform order">
    <p>
      <code>transform()</code> runs before <code>validate()</code>. The stored
      value is the transformed value when validation passes.
    </p>
  </Accordion>
  <Accordion title="Summary values">
    <p>
      Password prompts summarize as masked text, confirm prompts summarize as
      <code>yes</code> or <code>no</code>, list and multiselect prompts join
      values with commas, and date prompts summarize with their configured
      format.
    </p>
  </Accordion>
  <Accordion title="Non-interactive runs">
    <p>
      In non-TTY environments, prompts use matching flags, defaults, or
      <code>undefined</code> for optional prompts. If none of those exist,
      <code>oscli</code> exits with usage code <code>2</code>.
    </p>
  </Accordion>
</Accordions>
